Alternative Layouts in Shopify

Du wirst wahrscheinlich die Layout Datei theme.liquid kennen. Auf ihr basiert fast jede Seite in deinem Onlineshop.
Die meisten Developer verbringen die meiste Zeit damit Sections Snippets und Templates zu programmieren, deren Code innerhalb von theme.liquid verpackt ist.
Der Vorteil davon ist, dass durch das hier eingebundene zentrale CSS File ein konsistentes Layout über alle Seiten hinweg erzeugt wird.
Weiterhin führt es dazu, dass globale Layout Änderungen sehr leicht vorgenommen werden können und das einmal erstellt Layout wiederverwendet werden kann

Dieses Konzept stößt allerdings an seine Grenzen, wenn Seiten erschaffen werden sollen, die ein komplett anderes Designprinzip verfolgen
Viele Developer versuchen jetzt mit Custom CSS das vorherige Layout zu überschreiben. Dies ist unsauber und es gibt eine viel bessere Lösung

Ein häufiges Beispiel

Du wirst dich vielleicht fragen, in welchem Fall so ein neues Template hilfreich sein kann
Ein häufiges Beispiel sind Landingpages, da diese mit ihrem speziellen Design den Kunden sofort zur Kaufaktion bewegen sollen
Außerdem ist eine Werbeaktion oft der einzige Weg wie ein Kunde auf diese Seiten kommen kann.
Sie operieren also meist komplett unabhängig von den anderen Seiten des Onlineshops.

Erstellen eines neuen Layouts

Der Prozess ist sehr einfach.
Als erstes muss ein neue Datei mit erstellt werden, die im Layout Ordner abgespeichert wird.
Hier kann jetzt jeglicher HTML, CSS oder JS Code eingefügt werden, der auf allen Seiten sein soll, die das neue Layout verwenden.

Die folgenden Sachen müssen aber beachtet werden:

  • {{ content_for_header }} muss im Head Tag platziert werden. Dies sorgt dafür dass alle notwendigen Shopify Skripte hier inkludiert werden
  • {{ content_for_layout }} muss im Body Tag platziert werden. Dies sorgt dafür dass die Template Dateien hier dynamisch gerendert werden.
  • {% layout 'landing-page' %} muss die erste Zeile in der Template Datei der Landingpages heißen, die das neue Layout "landing-page.liquid" benutzen.

Weiterführende Gedanken

Manchmal kann es auch sehr nützlich sein überhaupt kein Layout File zu verwenden. Dies kann mit der ersten Zeile {% layout none %} in der Template Datei gelöst werden.
Dies kommt vorallem, dann zu Anwendung, wenn diese Seite komplett Standalone ist und das Layout auf keiner anderen Seite verwendet wird.

Es ist allerdings sehr wichtig trotzdem dem DRY treu zu bleiben.
Dafür erweisen sich in Shopify sogenannte Snippets sehr vorteilhaft. Sie können durch den Befehl {% include snippet %} sehr einfach eingebunden werden und erlauben eine simple Wiederverwendung von Code

Abschließend gilt zu sagen, dass auch wenn man es mit Alternative Layouts nicht übertreiben sollte, jeder Developer dieses Konzept immer im Hinterkopf haben muss.
